# E-commerce monorepo

pnpm workspace with three packages:

| Package | Path | What it is |
|---|---|---|
| `@e-commerce/client` | [apps/client](apps/client) | React 19 app with SSR via TanStack Router on Rsbuild |
| `@e-commerce/server` | [apps/server](apps/server) | Fastify 5 API (Clean Architecture, CQRS, Postgres) |
| `@e-commerce/storybook` | [apps/storybook](apps/storybook) | Storybook (rsbuild builder) for client components |

## Setup

```bash
pnpm install
```

Full step-by-step instructions (prerequisites, database, every run mode, troubleshooting): [docs/runbook.md](docs/runbook.md).

Requires Node >= 24 (`nvm use 24.18`); pnpm is pinned via the `packageManager` field (corepack fetches it automatically).

## Commands (from the repo root)

```bash
pnpm dev              # FULL STACK: client SSR dev server (5173) + Fastify API (4000), one command
                      #   client proxies /api/* → API; needs Postgres up (docker compose in apps/server)
pnpm dev:client       # client only → http://localhost:5173
pnpm build            # client production build (dist/ + dist/server/)
pnpm preview          # serve the client production build
pnpm test             # client unit tests (rstest)
pnpm check            # ALL checks, all packages, concurrent (~3-4s):
                      #   client: biome + tsc + rstest | server: biome + tsc + unit tests | storybook: build
pnpm check:client     # client checks only
pnpm e2e              # Playwright e2e (starts DB + API + client itself; needs Docker)
pnpm api:check        # server checks only
pnpm storybook        # storybook dev server → http://localhost:6006
pnpm build-storybook  # storybook static build
pnpm api              # Fastify API in watch mode (needs .env + Postgres, see apps/server)
```

Each package can also be driven directly: `pnpm --filter @e-commerce/<name> <script>`.

## Client SSR architecture

- [apps/client/src/router.tsx](apps/client/src/router.tsx): creates the router, shared by server and client. Client asset URLs are passed in via router context.
- [apps/client/src/routes/](apps/client/src/routes/): file-based routes; `src/routeTree.gen.ts` is generated by `@tanstack/router-plugin`.
- [apps/client/src/routes/__root.tsx](apps/client/src/routes/__root.tsx): renders the full HTML document. Its loader exposes the asset lists so they are dehydrated to the client and hydration matches.
- [apps/client/src/entry-server.tsx](apps/client/src/entry-server.tsx): streams HTML per request (`createRequestHandler` + `renderRouterToStream`).
- [apps/client/src/entry-client.tsx](apps/client/src/entry-client.tsx): hydrates the document with `RouterClient`.
- [apps/client/ssr/dev.mjs](apps/client/ssr/dev.mjs): Express dev server using Rsbuild in middleware mode.
- [apps/client/ssr/prod.mjs](apps/client/ssr/prod.mjs): Express production server; loads `dist/server/index.js` and reads asset URLs from `dist/manifest.json`.

Route loaders run on the server for the initial request (their data is dehydrated to the client), and on the client for subsequent navigations.

## API server

`apps/server` is a Fastify boilerplate with its own tooling (dbmate migrations, docker-compose for Postgres, Cucumber e2e). See [apps/server/README.md](apps/server/README.md). REST routes are prefixed with `/api`; default port 4000 is expected by the upcoming client integration (see the migration plan).
